[연구 일지] 2026-03-09

1. 주요 내용
- 무중단 배포(CI/CD) 파이프라인 아키텍처 분석 및 Git 커밋 해시를 활용한 롤백 태깅 자동화 적용.
- 인프라 최적화 환경에서 1만 트래픽 부하 테스트 성공 (오토스케일링/서버 다운 방어 검증).
- 테스트 중 발생한 대기열 역주행 및 우회 접속(새로고침) 취약점 식별 및 로직 수정 완료.

2. 진행 요약
- Cloud Build 트리거(mervis-auto-deploy)를 통한 CI(도커 빌드 및 업로드)와 인스턴스 템플릿 갱신을 통한 CD(MIG 롤링 업데이트)의 분리된 동작 원리를 명확히 분석함.
- 기존의 도커 이미지 `latest` 단일 태그 관리 방식이 장애 복구 지연을 초래함을 인지하고, Git 커밋 해시값만 이미지 태그로 기록하도록 파이프라인을 고도화함. 이를 통해 버그 발생 시 특정 해시값으로 즉각적인 롤백이 가능해짐.
- 1만 명 규모의 접속을 유발하는 부하 테스트를 실시하여 오토스케일링 및 알람 정책이 정상 작동함을 확인함. 최대 응답 시간 기준 500 에러: 2.2초, 400 에러: 0.328초, 200 정상: 1849초를 기록하며 서버 다운 없이 방어에 성공했으며, 테스트 종료 후 인스턴스 축소(Scale-in)까지 안정적으로 진행됨.
- 부하 테스트 모니터링 중, 대기열 숫자가 감소하다 다시 증가하는 현상과 새로고침 시 대기열 후순위로 밀리지 않고 바로 홈페이지로 접속되는 치명적 버그를 발견하여 즉시 수정함.

3. 주요 수정 및 개발 내용
[CI/CD 파이프라인]
- cloudbuild.yaml: 도커 이미지 빌드 시 태그에 `latest`를 제외하고 오직 Git 커밋 해시값만 기록되도록 설정 변경. 신규 템플릿 생성 시 해당 해시를 지정하여 롤링 업데이트 지시 구조 확립.

[Backend 최적화]
- 대기열 우회 차단: 사용자의 새로고침 동작 시 대기열 순서가 무시되는 결함을 수정하여, 비정상적인 세션 갱신에도 대기열 순서가 유지되거나 후순위로 재할당되도록 로직 강화.

4. 시행 착오 및 오류 해결
[자동 배포 파이프라인(CD) 오작동 분석]
- 현상: CI 작업(이미지 빌드 및 저장) 완료 후 자동으로 배포가 이루어지지 않는 현상 분석.
- 분석 및 해결: MIG의 롤링 배포는 '인스턴스 템플릿'이라는 설계도가 갱신될 때만 작동함을 확인함. 이미 설정 파일에 갱신 명령어는 존재했으나, `latest` 태그 관리 방식의 복구 취약점을 추가로 발견함. 이에 따라 Git 커밋 해시값만 기록하는 방식으로 변경하여 장애 발생 시 명확한 시점으로 템플릿을 되돌릴 수 있는 복구 능력을 확보함.

[1만 트래픽 부하 테스트 - 대기열 역주행 및 우회 버그]
- 현상: 트래픽 자체는 서버가 견뎌냈으나, 대기열 숫자가 오르내리기를 반복하고 새로고침을 통한 우회 접속이 가능한 문제 발생.
- 해결: 대기열 검증 로직을 전면 수정하여, 어떠한 접근 방식(새로고침 등)에도 철저히 대기열 후순위 배치 원칙이 적용되도록 결함을 해결함.
